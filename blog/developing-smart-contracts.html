Developing Smart Contracts

This is a collection of resources and notes on getting started with the development of smart contracts.

The Development Environment

Definitions and Preliminaries

The blockchain is a public database of records called blocks. When a transaction is made, it is stored in a block. A chain of blocks is made when transactions begin to relate to each other. A block should reference a parent block, meaning that information stored in one block cannot change without changing the remaining blocks. This means that we can verify transactions relatively easily.

Each block represents a transfer of cryptocurrency from one user to another in a network. More generally, a blockchain consists of the following (as stated in wikipedia):

  • Network of computers
  • Concensus mechanism
  • Blocks
  • Decentralized Applications or Smart Contracts

Any block has some tokens, a "nonce", a reference to the previous block's hash, and a hash of its own. The initial block in the blockchain is called the genesis block.

I use "Etherium" and "Etherium blockchain" interchangeably throughout these notes.

Concensus Mechanisms

The concensus mechanism exists to ensure that every computer on the network agrees with the creation of a new block and the state of the entire blockchain. Computers in the network are called nodes.

An user that wishes to create new blocks in the blockchain must solve a difficult math problem (of guessing the nonce of a block) to prove that they did the work required to make a transaction. The process of solving this problem is called mining, and miners are rewarded with Ether (the cryptocurrency of Etherium).

Each node (peer node, peer) has a copy of the blockchain. By checking the hash of the most recent block in the blockchain, the expected state of the blockchain can be stored and verified.

Etherium Accounts

An Etherium account is a type of entity existing in the Etherium blockchain. Any account has the ability to send and recevier Ether, each with an associated balance variable that tracks the current balance of the account.

An externally-owned account is authenticated by a private key, and a smart contract (contract) is a program that runs on the blockchain. The primary difference between the two account types is that a smart contract can only send transactions after it receives a transaction. One way to think of this is that it is an "API" that is called via transactions.

A smart contract is a program that runs on the blockchain. In other words, they are etherium accounts that are controlled by a program written by the user. Actual users can send transactions to the smart contract, allowing the smart contract to execute some functions and make resultant transactions.

Like a contract in real life, smart contracts can enforce some rules for the transaction that must be satisfied in order for the transaction to succeed.

There are two languages for building smart contracts:

  • Solidity
  • Vyper
