Linear Algebra Reference

This post is an overview of linear algebra. No prior knowledge in linear algebra is assumed, but trying to learn linear algebra from only this post is nearly impossible. It is more of a review than anything, or as a reference while learning the subject.

Linear algebra is the study of vector spaces and linear maps. We discuss their properties and representations. There are countless applications in various subjects in computer science, engineering, physics, and (naturally) mathematics. Linear algebra has been cleverly applied to other seemingly unrelated fields as well. To see the power of this subject, it may be worth looking into Google's PageRank Algorithm.

This treatment leaves out intuition, pictures, and other pedagogical tools that are very important for studying this subject. Nevertheless, the absolute prerequisite is elementary set theory. Single variable calculus is also recommended, but most content can be done without it. For a review of integration, see my post on integrals. Only the first section, titled "Understanding Integrals" is relevant.

Many things I want to write about require linear algebra, so I wrote this primarily for people to use it as a reference. The content is based on the books Linear Algebra Done Right by S. Axler and Linear and Geometric Algebra by A. Macdonald.

How To Use This Post

This post contains both exercises and explanations. Computational techniques, like row reduction, are not at all included. Many people like to provide visuals to represent objects in linear algebra. This is certainly beneficial, but it has not been done here. In my opinion, too much visualization defeats the purpose of linear algebra. It is algebra. I recommend that that you draw your own visuals to represent your understanding.

There are exercises in each section that you should attempt as soon as you see it. Most exercises ask you to prove or disprove some proposition.

Some exercises are marked as "optional." They are truly optional. The ones not marked as optional are essential. Without completing them, or at least looking through the solutions if you are doing a quick read, you will not gain a full understanding of the content. This style was adopted from Alan Macdonald's texts referenced above.

If you plan to do the exercises, do not look at the solution until you are stuck at a specific place (i.e. you know what you have to prove but you are not sure of how to prove it) and you have no clue how to proceed. Thinking about the subject is what leads to understanding it. However, sometimes thinking furiously about one problem for a long time is not beneficial. Try moving on temporarily or asking for hints (if one is not already provided) if possible.

Some (very few) exercises require working knowledge of some programming language. It asks that you implement some important algorithm.

Vectors

You may have been introduced to the idea behind a "vector" in a general math or physics course. Often, vectors are considered quantities that contain both a "magnitude and a direction," or sometimes "lists of numbers." Both of these definitions may be a sufficient introduction to understanding how vectors are represented, but not necessarily what they are.

Linear algebra axiomatizes the idea of a vector by defining what is called a vector space. It is an abstract mathematical object, but the abstractness of linear algebra is also what makes the subject so powerful.

Vector Spaces

We can start by developing some definitions that shape mathematical objects and proving properties of these objects. This will be done many times throughout linear algebra and other areas of mathematics.

Subspaces

  1. (u1+u2)∈U(\mathbf{u}_1 + \mathbf{u}_2) \in \mathbf{U}(u1​+u2​)∈U (Closed under vector addition)
  2. au1∈Ua\mathbf{u}_1 \in \mathbf{U}au1​∈U (Closed under scalar multiplication)

Note that in order for U\mathbf{U}U to be closed under scalar multiplication, the zero vector must also be in U\mathbf{U}U.

∑i=1naivi\sum_{i=1}^n a_i\mathbf{v}_ii=1∑n​ai​vi​

Linear Independence, Bases, and Dimension

Intuitively, we say that a set of vectors is linearly independent if no vector in the set can be represented as a linear combination of the other vectors. Formally, a set of vectors v1,…,vn\mathbf{v}_1, \dots, \mathbf{v}_nv1​,…,vn​ is linearly independent if and only if for scalars {ai}\{a_i\}{ai​}, the only way to satisfy

a1v1+⋯+anvn=0a_1\mathbf{v}_1 + \cdots + a_n\mathbf{v}_n = \mathbf{0}a1​v1​+⋯+an​vn​=0

is to set ai=0a_i = 0ai​=0 for every iii.

To better understand the intuitive explanation given before, try to write v1\mathbf{v}_1v1​ as a linear combination of the other vectors:

a1v1+⋯+anvn=0a_1\mathbf{v}_1 + \cdots + a_n\mathbf{v}_n = \mathbf{0}a1​v1​+⋯+an​vn​=0
a1v2=−a2v2−⋯−anvna_1\mathbf{v}_2 = -a_2\mathbf{v}_2 - \cdots - a_n\mathbf{v}_na1​v2​=−a2​v2​−⋯−an​vn​

Dividing by a1a_1a1​ is impossible because a1a_1a1​ must be equal to 000 for this to be true. The same argument can be made for v2,…,vn\mathbf{v}_2, \dots, \mathbf{v}_nv2​,…,vn​.

Bases

A basis for a subspace U\mathbf{U}U is any linearly independent set of vectors that span U\mathbf{U}U. In Exercise 1.8, the set of vectors SSS is a basis for U\mathbf{U}U. The same exercise also proves that any basis has exactly nnn vectors.

This is a phenomenal development. The idea of a basis is essential and will come up many times both in and outside of linear algebra because it allows us to refer to a specific vector in a subspace without reference to a coordinate system, i.e. we can refer to a vector as a mathematical object rather than in terms of its coordinates. We will discuss the former in the next section, but the definition of "dimension" is provided below.

Inner Product Spaces

We now define what is known as the "inner product." More familiar to you already may be the "dot," or "scalar" product. For two vectors u\mathbf{u}u and v\mathbf{v}v, the dot product is denoted u⋅v\mathbf{u} \cdot \mathbf{v}u⋅v. Let

E={(1,…,0),…,(0,…,1)}.\cal{E} = \{(1, \dots, 0), \dots, (0, \dots, 1)\}.E={(1,…,0),…,(0,…,1)}.

If u,v∈Rn\mathbf{u}, \mathbf{v} \in \mathbf{R}^nu,v∈Rn and have coordinates (u1,…,un)E(u_1, \dots, u_n)_{\cal{E}}(u1​,…,un​)E​ and (v1,…,vn)E(v_1, \dots, v_n)_{\cal{E}}(v1​,…,vn​)E​ respectively, then the product can be computed using:

u⋅v=u1v1+⋯+unvn(2.1)\mathbf{u} \cdot \mathbf{v} = u_1v_1 + \cdots + u_nv_n \tag{2.1} \label{2.1}u⋅v=u1​v1​+⋯+un​vn​(2.1)

This is not a useless formulation. For example, for a constant force F\mathbf{F}F acting on a mechanical system over a line of motion r\mathbf{r}r, the work (a scalar) turns out to be W=F⋅rW = \mathbf{F} \cdot \mathbf{r}W=F⋅r.

(2.1)\eqref{2.1}(2.1) gives an efficient way to compute the result of this operation. However, in linear algebra we are much more general. The power of generality is demonstrated well here.

Definitions

An inner product space is an extension of a vector space V\mathbf{V}V (it doesn't need to be finite dimensional!) over a field F\mathbf{F}F. It has an operation T:V×V→FT : \mathbf{V} \times \mathbf{V} \to \mathbf{F}T:V×V→F. Usually T(u,v)T(\mathbf{u}, \mathbf{v})T(u,v) is written as ⟨u,v⟩\langle \mathbf{u}, \mathbf{v} \rangle⟨u,v⟩ or sometimes u⋅v\mathbf{u} \cdot \mathbf{v}u⋅v. I denote it as the latter, because the former takes much longer to type.

The map TTT will come up later, but it also must follow some properties:

  1. u⋅v=v⋅u\mathbf{u} \cdot \mathbf{v} = \mathbf{v} \cdot \mathbf{u}u⋅v=v⋅u
  2. (u)⋅v+w=u⋅v+u⋅w(\mathbf{u}) \cdot \mathbf{v} + \mathbf{w} = \mathbf{u} \cdot \mathbf{v} + \mathbf{u} \cdot \mathbf{w}(u)⋅v+w=u⋅v+u⋅w
  3. au⋅v=a(u⋅v)a\mathbf{u} \cdot \mathbf{v} = a(\mathbf{u} \cdot \mathbf{v})au⋅v=a(u⋅v)

Linear Maps

Linear maps are sometimes also called linear transformations.

Without an understanding of linear maps, the study of linear algebra would be incomplete. You will see these in many areas of mathematics, such as (abstract) algebra, analysis, physics, etc.

Definition and Examples

Consider a function f:U→Vf : \mathbf{U} \to \mathbf{V}f:U→V for vector spaces U\mathbf{U}U and V\mathbf{V}V. We say that the domain of fff, denoted dom(f)\text{dom}(f)dom(f), is U\mathbf{U}U. The codomain of the function is V\mathbf{V}V.
